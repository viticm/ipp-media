Overview

This sample demonstrates how to use the Intel ® Integrated Performance Primitives (Intel ® IPP) to create an H.263 encoder. Currently H.263 decoder supports baseline profile plus Annexes D, F, I  of the ITU-T Recommendation H.263 (01/05).

Additional information on this software as well as other Intel software performance products is available at http://developer.intel.com/software/products/ .

Description

Apart from the UMC::VideoEncoder the H.263 encoder provides additional methods for RFC2190 packetization and MSE calculation

  • int GetFrameMacroBlockPerRow() - returns the number of MBs per row
  • int GetFrameMacroBlockPerCol() - returns the number of MBs per column
  • Ipp8u* GetFrameQuant() - returns the pointer to the internal array with quant for each MB in the frame
  • IppMotionVector* GetFrameMVpred() - returns the pointer to the internal array with MV predictors for each MB in the frame
  • Ipp32u* GetFrameMBpos() - returns the pointer to the internal array with bit position for each MB in the frame
  • void GetFrameMSE(Ipp64s *y, Ipp64s *u, Ipp64s *v) - returns MSE for the frame for Y, U and V components

Known Limitations

The decoder supports the standard ITU-T Recommendation H.263 (01/05) with certain restrictions. Only the following features are supported in the current version:

  • I-Frames
  • P-Frames
  • Unrestricted MV (Annex D)
  • Advanced prediction with 4MV and OBMC (Annex F)
  • Advanced Intra Prediction (Annex I)
  • Standardized H.263 picture formats, CIF  picture clock frequency
